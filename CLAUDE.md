## Overview

This is a Cargo workspace providing safe Rust bindings to the [SUNDIALS](https://computing.llnl.gov/projects/sundials) ODE/DAE solver library. It contains two crates:

- **`sundials-rs-sys`** â€” raw, unsafe FFI bindings generated by `bindgen` from `sundials-rs-sys/wrapper.h`. Most users should not touch this directly.
- **`sundials-rs`** â€” safe, idiomatic Rust wrapper. This is the primary crate. Modules: `cvode`, `cvodes`, `ida`, `idas`, plus internal helpers (`context`, `nvector`, `matrix`, `linear_solver`, `error`).

## Commands

```bash
# Build (from workspace root)
cargo build

# Build with vendored SUNDIALS (downloads + compiles SUNDIALS 7.4.0 via CMake; no system lib needed)
cargo build --features sundials-rs-sys/vendored

# Run tests
cargo test

# Run a single example
cargo run --example robertson_cvode

# Check (no codegen, faster)
cargo check

# Lint
cargo clippy

# Generate docs (opens in browser)
cargo doc --open
```

If SUNDIALS is installed in a non-standard location:
```bash
export SUNDIALS_DIR=/path/to/sundials/install
```

## Architecture

### Layer separation

`sundials-rs-sys` â†’ `sundials-rs`

`sundials-rs-sys/src/lib.rs` simply includes `bindings.rs` generated at build time from `wrapper.h` via `bindgen`. The build script (`build.rs`) either runs `pkg-config` to locate the system library, or (with `vendored` feature) downloads, patches, and compiles SUNDIALS with CMake.

`sundials-rs/src/` wraps the raw bindings in safe types. Each solver module (`cvode.rs`, `cvodes.rs`, `ida.rs`, `idas.rs`) follows the same pattern:
1. A `*Builder` struct configures the solver via method chaining.
2. `.build(closure)` consumes the builder and returns a `*Solver<F>` holding the SUNDIALS memory and all SUNDIALS N_Vectors, matrices, and linear solver in a single owned struct.
3. Solver methods (`step`, `reinit`, `stats`, `sensitivities`, `calc_ic`) call through to SUNDIALS C functions via `sundials_rs_sys`.

### CVODES / IDAS sensitivity pattern

For forward sensitivity (FSA), the RHS/residual closure must read parameters through a raw pointer (`builder.params_ptr()`) captured **before** `build()` consumes the builder. This allows SUNDIALS's internal difference-quotient (DQ) approximation to temporarily perturb parameter values and re-evaluate the RHS. Hardcoded Rust constants will produce zero sensitivities silently.

### Borrow checker interaction with `step()`

`step()` returns `&[f64]` borrowed from inside the solver. If `sensitivities()` or any other `&mut self` method is needed after stepping, the returned slice must be copied to a `Vec<f64>` first to release the borrow.
